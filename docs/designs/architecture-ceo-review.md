## Vision

### 10x Check
The 10x version of RustyClinic is not just an EMR — it's the deployment platform for digital health in low-resource settings. The package system becomes an app store for health programs. The fleet management becomes a national health IT backbone. The training mode becomes a health worker competency system. The architecture already points in this direction; the expansions below make it explicit.

## Architectural Decisions from Review

### Resolved
1. **Inter-domain crate dependency rules**: Explicit rules required. Domain crates must declare which other domain crates they depend on. Circular deps disallowed.
2. **Device security lifecycle**: Added to architecture — device registration, revocation, remote wipe, full-disk encryption requirements, stolen-device response.
3. **Queue claim/lock mechanism**: Add assigned_to field and optimistic lock on the `called` transition in the queue state machine.
4. **Audit integrity violation response**: Define detection, alert, quarantine, and repair procedure for broken audit chains.
5. **Observability architecture**: Add structured logging format, trace ID propagation, key metrics, and per-tier alerting strategy.

### Deferred
1. **LLM prompt injection defense**: Deferred to AI design doc (Phase 9).
